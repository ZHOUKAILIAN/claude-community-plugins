# AI 文档驱动开发插件

将任何项目转换为文档驱动开发工作流程，提供AI辅助支持。此插件自动检测、生成和维护项目文档标准，确保一致且可维护的开发实践。

## 概述

AI文档驱动开发插件帮助团队建立和维护文档优先的开发工作流程。它强制执行所有代码变更必须先进行相应文档更新的原则，从而创建更好的可维护性和团队协作。

## 核心功能

- **🔧 CLAUDE.md强制执行**: 自动确保CLAUDE.md包含强制性的文档驱动开发工作流程
- **📊 文档分析**: 全面评估项目文档完整性
- **🎯 模式提取**: 自动提取项目特定的编码模式和约定
- **📝 智能生成**: 基于项目分析和模板创建标准化文档

## 包含的技能

### 1. claude-md-enforcer
在CLAUDE.md文件中强制执行文档驱动开发工作流程。此技能优先运行，为文档优先开发建立基础。

**使用场景:**
- 在新项目中初始化文档驱动开发
- 更新现有项目以遵循文档标准
- 使CLAUDE.md与不断发展的项目标准保持同步

### 2. doc-detector
分析项目文档完整性并识别缺失的关键文档。

**使用场景:**
- 评估当前文档状态
- 识别文档覆盖范围的差距
- 为文档改进准备建议

### 3. pattern-extractor
从现有代码库中提取项目特定的编码模式、约定和标准。

**使用场景:**
- 记录现有项目编码标准
- 为新团队成员创建指南
- 确保AI代码生成遵循既定模式

### 4. doc-generator
使用智能模板和项目分析生成标准化项目文档。

**使用场景:**
- 创建doc-detector识别的缺失文档
- 建立全面的文档结构
- 基于提取的模式生成项目特定标准

## 开始使用

1. **安装插件**: 将此插件添加到您的Claude Code环境
2. **运行初始设置**: 使用claude-md-enforcer技能建立文档驱动开发工作流程
3. **分析项目**: 使用doc-detector了解当前文档状态
4. **提取模式**: 使用pattern-extractor记录现有编码约定
5. **生成文档**: 使用doc-generator创建缺失的文档

## 文档结构

插件建立并维护以下标准文档结构：

```
docs/
├── requirements/     # 项目需求和规范
├── design/          # 技术设计和架构
├── analysis/        # 项目和代码库分析
├── standards/       # 编码标准和约定
└── [自定义]/        # 项目特定文档
```

## 工作流程集成

此插件与现有开发工作流程无缝集成：

1. **代码变更前**: 更新docs/中的相关文档
2. **AI开发**: Claude Code CLI读取docs/获取上下文和约束
3. **代码实现**: 基于文档化的需求和标准实现变更
4. **审查流程**: 确保代码与文档化的规范匹配

## 最佳实践

- **从CLAUDE.md开始**: 总是首先运行claude-md-enforcer以建立工作流程标准
- **定期分析**: 定期运行doc-detector以维护文档完整性
- **模式更新**: 当发生重大架构变更时重新运行pattern-extractor
- **模板定制**: 将生成的文档模板适配到项目特定需求

## 优势

- **一致性**: 确保所有团队成员遵循相同的开发标准
- **可维护性**: 为长期项目维护创建全面的文档
- **AI效果**: 为更准确的AI辅助开发提供丰富的上下文
- **团队入职**: 通过清晰的文档简化新团队成员的集成
